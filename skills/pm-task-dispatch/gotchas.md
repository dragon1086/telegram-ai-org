# PM Task Dispatch — Gotchas

## 1. 복합 태스크의 의존성 순서
"디자인 후 개발" 같은 태스크는 반드시 순서대로 배분.
병렬 실행 가능 여부를 먼저 판단하고, 의존성이 있으면 순차 배분.

## 2. 기본 담당봇 오판
"API 문서 작성"은 개발실이 아닌 제품실(기획)이 더 적합할 수 있음.
키워드 매칭만으로 판단하지 말고 태스크 전체 맥락을 읽을 것.

## 3. 배분 후 추적 누락
배분했다고 끝이 아님. 완료 예상 시간 후 팔로업 메시지 필수.
`config.json`의 `default_timeout_hours` 기준으로 체크.

## 4. chat_id 하드코딩 주의
`config.json`의 `chat_id`를 수정 없이 사용하면 잘못된 그룹에 메시지 전송.
실행 전 반드시 `TELEGRAM_GROUP_CHAT_ID` 환경변수 확인.

## 5. 봇 오프라인 상태 미확인
배분 전 봇이 응답 가능한지 확인 안 하면 태스크가 사라짐.
`/harness-audit` 스킬로 봇 상태 먼저 점검 권장.
