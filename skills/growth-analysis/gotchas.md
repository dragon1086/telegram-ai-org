# Growth Analysis — Gotchas

이 스킬을 사용할 때 자주 발생하는 실수와 주의사항이다.

## Gotcha 1: 데이터 없이 지표 현재값을 추정값으로 채움
**상황**: DAU, 전환율, 리텐션 등 실제 수집된 데이터가 없을 때 분석을 진행할 때
**증상**: "추정 DAU 약 100명" 같은 근거 없는 수치가 분석 문서에 포함됨. 이 수치가 이후 의사결정에 사실처럼 인용됨
**해결**: 데이터 소스 확인을 Step 0으로 추가. `logs/` 디렉토리와 `.ai-org/runs/` 에서 실제 실행 이력 확인 후 분석 시작. 데이터 없으면 "데이터 미수집 — 수집 방법 제안"을 분석 결과로 출력

## Gotcha 2: 전주/전월 대비 비교 시 기간 불일치
**상황**: 이번 주 5일치 데이터를 지난 주 7일치와 비교할 때
**증상**: "이번 주 완료 태스크 35개 vs 지난 주 49개, 29% 감소"로 보고하지만 실제로는 기간 차이로 인한 착시. 잘못된 트렌드 판단으로 이어짐
**해결**: 비교 기간을 분석 문서 상단에 명시 (예: "2026-03-10~16 vs 2026-03-03~09"). 동일 기간 단위로만 비교. 기간이 다르면 일별 평균으로 정규화

## Gotcha 3: 실험 제안이 실행 불가능한 수준으로 추상적
**상황**: A/B 테스트 아이디어를 "사용자 경험 개선 실험" 수준으로 기술할 때
**증상**: 개발실 봇이 실험을 어떻게 구현할지 알 수 없어 실험이 시작되지 않음. 다음 분석 주기에도 같은 권장사항이 반복됨
**해결**: 실험 제안 형식 표준화 — 가설: "X를 바꾸면 Y가 Z% 향상될 것", 측정 지표: 구체적 수치, 구현 범위: 예상 코드 변경 위치, 기간: N일. telegram-ai-org 맥락에서는 봇 응답 템플릿, 라우팅 로직 변경 등 구체 위치 명시

## Gotcha 4: docs/growth/ 디렉토리 미생성으로 파일 저장 실패
**상황**: 분석 보고서를 `docs/growth/YYYY-MM-DD-analysis.md`로 저장할 때
**증상**: `docs/growth/` 디렉토리가 없어 저장 실패. 오류 없이 조용히 넘어가면 분석 결과가 유실됨
**해결**: 저장 전 `docs/growth/` 디렉토리 존재 확인. 없으면 생성 후 저장. `.gitkeep` 추가 여부도 확인
