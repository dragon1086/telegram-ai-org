## Phase 2: 우열 3분류 및 결론

### 멀티에이전트가 명확히 우수한 항목

| 항목 | 내용 |
|------|------|
| 정확도 | 라이선스 Apache 2.0 정확 기재 (단일은 MIT 오기재 — 치명 오류) |
| 분석 깊이 | 릴리즈 히스토리 3버전 추적, 커뮤니티 정량(Stars/Forks/Issues), 2026 벤치마크 |
| 실무 활용성 | 4시나리오 도입 매트릭스, 리스크 5개+대응, 설치 코드, CVE 패치 정보 |
| 커버리지 | FinGPT·OpenBB 추가 경쟁사, FinRobot(Columbia대), TradeTrap 반론 논문 발굴 |

### 단일 에이전트가 우수한 항목

| 항목 | 내용 |
|------|------|
| 수치 구체성 | 누적수익률 23.21%, 연간 24.90%, Sharpe 5.60 직접 인용 (조직은 "원문 참조" 처리) |
| 간결성 | 6개 섹션, 빠른 파악 가능 |
| 일부 세부 기능 | v0.2.2 five-tier rating scale, effort control, OpenAI Responses API 지원 포착 |

### 차이가 미미한 항목

| 항목 | 내용 |
|------|------|
| 기본 아키텍처 | LangGraph 4계층 구조 — 양쪽 동일 기술 |
| 개발 주체/논문 | Tauric Research, arXiv:2412.20138 — 양쪽 동일 |
| 조사 기준 시점 | 2026-03-22 — 동일 |

### 결론을 뒤집을 수 있는 반론·한계

1. **내부 불일치가 신뢰성 훼손**: 기획실(Apache 2.0)↔개발실(MIT) 상충 — 조율 없으면 멀티에이전트가 오히려 혼선 제공
2. **평가 모델 주관성**: 5점 척도 기준 정의 없이 작성자가 임의 부여 — 재현성 낮음
3. **샘플 1개**: 단일 태스크 비교는 일반화 근거 부족
4. **성장 전략 공백**: 양쪽 모두 TAM/SAM/SOM, KPI 없음 — 기술 리서치 이외 목적엔 동등하게 부족

## Phase 3: 의사결정자용 최종 요약 보고서

### 결론

**복합 에이전트가 단일보다 실질적으로 우수하다 — 단, 조직 간 충돌 해소가 전제 조건이다.**

종합 점수 28/30 vs 20/30으로 +8점 우위. 특히 실무 의사결정에 직결되는
정확도·커버리지·실무 활용성에서 격차가 두드러진다.

---

### 핵심 근거 (5가지)

1. **정확도 오류 방지**: 단일 에이전트의 라이선스 MIT 오기재는 상업적 도입 검토 시 법적 오판을 유발할 수 있는 치명 오류. 멀티에이전트(기획실)는 Apache 2.0 정확 기재.
2. **커버리지 확장**: 단일이 누락한 FinRobot(Columbia대), TradeTrap 반론 논문, TraderBench 최신 벤치마크를 추가 발굴 — 경쟁 환경 파악의 폭이 다름.
3. **실무 즉시 활용**: 도입 판단 매트릭스(4시나리오), 리스크 5개+대응방안, 설치 코드 포함 — 단일의 "수치만 나열"과 달리 의사결정까지 지원.
4. **버전 추적 깊이**: v0.2.0→v0.2.1→v0.2.2 변화 이력(ChromaDB→BM25, CVE 패치)을 추적 — 유지관리 판단 근거 제공.
5. **역보완 가능성**: 단일의 논문 수치 직접 인용(23.21%, Sharpe 5.60)은 멀티에이전트가 역보완 가능한 약점 — 구조적 패배 요인이 아님.

---

### 이 결론이 유효한 조건

- PM이 태스크 배분 시 "실행 범위"를 명시하여 조직 간 중복·오버스콥이 없을 때
- 조직 간 결과 불일치(라이선스 등)가 PM에 의해 추가 조율 태스크로 발동될 때
- 동일 태스크를 2개 이상 조직이 독립 검증한 결과가 교차 검토될 때

### 추가 검증이 필요한 영역

- **라이선스 확정**: GitHub LICENSE 파일 직접 확인 (Apache 2.0 vs MIT 최종 판정)
- **LLM 모델명 정확성**: GPT-5.4, Gemini 3.1, Claude 4.6 등 단일 보고서 기재 모델명 실제 API ID와 일치 여부
- **평가 척도 표준화**: 5점 척도 기준 정의 없음 — 재현 가능한 루브릭 필요
- **다수 태스크 누적**: 1회 비교는 통계적 근거 부족 — 3~5회 반복 비교 후 패턴 확인 권장
