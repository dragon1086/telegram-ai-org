## 1. 시스템 아키텍처 개요

TradingAgents는 실제 트레이딩 회사의 조직 계층을 LLM 에이전트로 모델링한 **계층형 멀티 에이전트 파이프라인**이다.

```
┌─────────────────────────────────────────────────────────┐
│                    입력 레이어                            │
│   ticker (종목코드) + date (날짜)                        │
└────────────────────┬────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────┐
│              데이터 수집 레이어                           │
│  Alpha Vantage API → 주가·재무·뉴스·소셜 데이터           │
└────────────────────┬────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────┐
│             애널리스트 팀 (병렬)                          │
│  ┌─────────────┐ ┌────────────┐ ┌──────────┐ ┌────────┐│
│  │ Fundamentals│ │ Sentiment  │ │  News    │ │Technical│
│  │  Analyst    │ │  Analyst   │ │ Analyst  │ │Analyst  │
│  └──────┬──────┘ └─────┬──────┘ └────┬─────┘ └───┬────┘│
└─────────┼──────────────┼─────────────┼────────────┼─────┘
          └──────────────┴─────────────┴────────────┘
                                │
                    애널리스트 리포트 취합
                                │
┌───────────────────────────────▼─────────────────────────┐
│              리서처 팀 (토론/디베이트)                    │
│  ┌─────────────────┐     ┌───────────────────────────┐  │
│  │ Bullish Researcher│◄──►│ Bearish Researcher        │  │
│  └─────────────────┘     └───────────────────────────┘  │
│           max_debate_rounds 설정으로 라운드 조정          │
└───────────────────────────────┬─────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────┐
│                   트레이더 에이전트                       │
│        종합 분석 → 매수/매도/홀드 결정 + 규모 결정        │
└───────────────────────────────┬─────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────┐
│             리스크 관리 / 포트폴리오 매니저               │
│        리스크 평가 → 거래 승인/거부/조정                  │
└───────────────────────────────┬─────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────┐
│                  출력 (거래 결정)                         │
│    action (BUY/SELL/HOLD) + confidence + reasoning       │
└─────────────────────────────────────────────────────────┘
```

---

## 2. 오케스트레이션 방식: LangGraph

- **프레임워크**: LangGraph (LangChain 기반 상태 기계 그래프)
- **실행 방식**: 방향성 그래프(DAG) 기반 에이전트 라우팅
- **상태 관리**: `TradingAgentsGraph` 클래스에서 전체 상태 관리
- **설정 파일**: `default_config.py`에서 전체 파라미터 관리

---

## 3. 데이터 흐름

```
Alpha Vantage API
  ├── 주가 데이터 (OHLCV)
  ├── 재무 데이터 (Income Statement, Balance Sheet)
  ├── 뉴스 피드
  └── 소셜 미디어 감성 데이터
       │
       ▼
에이전트별 특화 분석
       │
       ▼
BM25 기반 메모리 모듈 (과거 분석 저장/검색)
       │
       ▼
디베이트 라운드 (설정 가능)
       │
       ▼
최종 거래 결정 + 리포트 저장 (하위폴더 구조)
```

---

## 4. 주요 설정 파라미터

| 파라미터 | 설명 | 기본값 |
|----------|------|--------|
| `llm_provider` | LLM 제공자 선택 | openai |
| `deep_think_llm` | 심층 분석용 모델 | gpt-5.2 |
| `quick_think_llm` | 빠른 판단용 모델 | gpt-5-mini |
| `max_debate_rounds` | 리서처 디베이트 최대 라운드 | 2 |
| `temperature` | LLM 창의성 조절 | 설정 가능 |

## 5. 메모리 아키텍처

- **v0.1.x**: ChromaDB 벡터 스토어 기반 유사도 검색
- **v0.2.0+**: BM25 (키워드 기반 검색) 교체 — 비OpenAI LLM 호환성 향상 목적
- **목적**: 과거 분석 리포트 저장 → 미래 분석 시 참조

## 6. 모듈 간 의존관계 요약

```
tradingagents/
  graph/trading_graph.py  ← 핵심 오케스트레이터
    ├── agents/analysts/   ← 4개 분석가 (독립 실행)
    ├── agents/researchers/ ← Bullish/Bearish 디베이트
    ├── agents/trader/     ← 종합 결정
    ├── agents/risk_mgmt/  ← 리스크 필터
    ├── agents/managers/   ← 포트폴리오 관리
    └── agents/utils/      ← 공통 도구 (API 호출, 파싱)
```
