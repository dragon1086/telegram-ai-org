## 1. 에이전트 전체 목록

| 에이전트 | 팀 | 역할 | 입력 | 출력 |
|---------|-----|------|------|------|
| Fundamentals Analyst | Analyst | 재무지표·내재가치 평가 | 재무제표, 밸류에이션 데이터 | 펀더멘털 리포트 |
| Sentiment Analyst | Analyst | 시장 감성 분석 | 소셜미디어, 투자자 심리 데이터 | 감성 리포트 |
| News Analyst | Analyst | 뉴스·매크로 이벤트 분석 | 글로벌 뉴스 피드 | 뉴스 리포트 |
| Technical Analyst | Analyst | 기술적 지표 패턴 분석 | OHLCV 데이터 (MACD, RSI 등) | 기술적 분석 리포트 |
| Bullish Researcher | Researcher | 상승 논거 구성·방어 | 4개 애널리스트 리포트 | 매수 근거 문서 |
| Bearish Researcher | Researcher | 하락 논거 구성·방어 | 4개 애널리스트 리포트 | 매도 근거 문서 |
| Trader Agent | Trader | 종합 판단 → 거래 결정 | 리서처 디베이트 결과 | 거래 지시 (BUY/SELL/HOLD + 규모) |
| Risk Manager | Risk Mgmt | 리스크 평가·승인·거부 | 트레이더 거래 지시 | 승인/거부/조정된 거래 |
| Portfolio Manager | Manager | 포트폴리오 전체 관리 | 리스크 매니저 승인 거래 | 최종 포트폴리오 상태 |

## 3. 에이전트 간 통신 방식

| 방식 | 설명 |
|------|------|
| **상태 그래프 (LangGraph)** | 각 에이전트는 공유 상태 객체에 결과를 기록 |
| **텍스트 리포트 전달** | 에이전트 간 통신은 자연어 리포트 형태 |
| **직접 메시지 패싱** | Bullish/Bearish 리서처는 서로의 논거를 직접 참조하며 반박 |
| **BM25 메모리** | 과거 세션 리포트를 검색해 현재 분석에 참조 |

## 4. 에이전트 페르소나 설정

- **Aggressive Trader** vs **Conservative Trader**: v0.2.0에서 "risky/safe" 명칭에서 변경
- **리서처 디베이트**: 구조화된 토론으로 확증 편향 방지
- **Risk Manager**: 트레이더 판단을 독립적으로 검증하는 체크포인트 역할
