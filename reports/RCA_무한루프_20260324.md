## 결론

**근본 원인은 `pm_orchestrator.py:on_task_complete()`의 두 가지 버그가 결합된 구조적 결함이다.**

1. `_TERMINAL` 집합에 `needs_review` 누락 → parent가 이미 `needs_review` 상태여도 재트리거 가능
2. `_synthesize_and_act()` 호출 전 **parent 현재 상태 무확인** → 이미 처리된 parent를 반복 재합성

이 두 버그가 T-362의 CONFLICTING 판정과 맞물려 **5.5시간 312회 동일 LLM 합성 반복**을 유발했다.

## Phase 2 산출물 — 근본원인 분석 보고서

### 버그 1 (PRIMARY) — `_TERMINAL`에 `needs_review` 누락

**위치**: `core/pm_orchestrator.py`, 함수 `on_task_complete`, **라인 1304**

```python
# 현재 코드 (버그)
_TERMINAL = {"done", "failed", "cancelled"}   # ← needs_review 빠짐

# 결과:
# parent_status == "needs_review"여도 이 집합에 없음
# → 서브태스크 all_done 체크는 통과
# → _synthesize_and_act 재실행됨
```

`needs_review`는 합성 결과가 CONFLICTING일 때 parent에 부여되는 상태다.
이 상태는 "사람의 확인이 필요한 종료 상태"이므로 terminal로 취급해야 하지만,
`_TERMINAL` 집합에서 빠져 있어 무한 재진입이 가능한 구조를 만들었다.

---

### 버그 2 (PRIMARY) — parent 현재 상태 무확인

**위치**: `core/pm_orchestrator.py`, 함수 `on_task_complete`, **라인 1325~1326**

```python
# 현재 코드 (버그)
if all_done:
    await self._synthesize_and_act(parent_id, siblings, chat_id)
    # ↑ parent가 이미 needs_review / done / failed 상태여도 무조건 재합성

# 필요한 가드 (현재 없음):
# parent_task = await self._db.get_pm_task(parent_id)
# if parent_task["status"] in (_TERMINAL | {"needs_review"}):
#     return  # 이미 처리된 parent — 재합성 불필요
```

서브태스크 `all_done` 여부만 확인하고, **parent가 이미 처리 완료 상태인지 확인하는 코드가 전혀 없다.**
이 때문에 heartbeat/poller 사이클마다 동일 서브태스크 조합으로 재트리거된다.

**위치**: `core/pm_orchestrator.py`, 함수 `_synthesize_and_act`, **라인 1671~1700**

```python

    parent = await self._db.get_pm_task(parent_task_id)
    original_request = parent["description"][:1500] if parent else ""
    # ↑ parent 상태 읽어오지만 terminal 여부를 체크하지 않고 즉시 LLM 합성 진행
    # → 방어적 가드(idempotency guard)가 없음
```

`_synthesize_and_act`는 함수 진입 시 parent 상태를 가져오지만,
이미 `done`/`needs_review`/`failed` 상태일 때 조기 반환하는 로직이 없다.
버그 1·2가 수정되더라도 이 함수 자체가 재진입에 취약한 구조다.

## 재발 방지 요약

이번 무한루프는 **외부 장애가 아닌 내부 상태 기계(state machine) 설계 결함**이다.
`needs_review`가 사실상 terminal 상태임에도 코드 상 terminal로 처리되지 않아,
정상 동작하는 heartbeat/poller 사이클이 매분 같은 synthesis를 재실행시켰다.
P0 2건 수정만으로 이번 재발은 완전히 차단 가능하다.
