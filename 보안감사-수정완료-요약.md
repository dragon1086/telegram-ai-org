## 1. Phase 1 — 보안 파일 수정

### ① Dockerfile: secret-like ENV 제거 및 런타임 주입 구조 전환

| 조치 | 내용 |
|------|------|
| **제거된 ENV 변수** | `PM_BOT_TOKEN`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_GROUP_CHAT_ID`, `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GEMINI_API_KEY`, `CLAUDE_CODE_OAUTH_TOKEN` |
| **유지된 ENV 변수** | 경로/타임아웃/기능플래그 등 비민감 설정값 |
| **런타임 주입 방식** | `docker run --env-file .env` 또는 `docker-compose env_file` |
| **이유** | Dockerfile에 secret 변수명이 있으면 이미지 레이어 히스토리 분석 시 노출 가능 — 이름만 있어도 공격자에게 변수 목록 힌트 제공 |

변경 전:
```dockerfile
ENV PM_BOT_TOKEN="" \
    TELEGRAM_BOT_TOKEN="" \
    ANTHROPIC_API_KEY="" \
    ...
```

변경 후 (주석으로 런타임 주입 안내 교체):
```dockerfile
# !! 런타임 주입 전용 — docker run --env-file .env 사용
# !! 주입 필수 변수: PM_BOT_TOKEN, TELEGRAM_BOT_TOKEN, ANTHROPIC_API_KEY ...
ENV CLAUDE_CLI_PATH="/opt/cli/bin/claude" \
    ...
```

### ② .gitignore — 민감 파일 패턴 추가

| 추가된 패턴 | 용도 |
|------------|------|
| `*.pem` | SSL/TLS 인증서 |
| `*.key` | 개인키 파일 |
| `*.crt` | 공개 인증서 |
| `*.p12` | PKCS#12 키스토어 |
| `secrets/` | 시크릿 디렉토리 |
| `.env.*` | 환경별 env 파일 (`.env.production`, `.env.staging` 등) |

> 기존 `.env`, `*.env` 패턴은 유지, 위 6개 패턴 누락분만 추가

| 추가된 패턴 | 용도 |
