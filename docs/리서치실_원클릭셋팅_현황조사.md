## 결론 (한 줄)

**지금 상태로는 우분투 원클릭 E2E 테스트가 "거의" 가능하지만, Node.js·Python 3.11+ 자동 설치 단계 누락 + 봇 토큰 7개 수동 발급이라는 2개 blocker가 있다.**

## 최종 종합 판단: 원클릭 E2E 가능한가?

### 질문 1: "우분투에서 원클릭으로 E2E 테스트 가능한가?"

**현재 상태**: 부분 가능. 구체적으로:

```
✅ 가능한 것: 로컬 mock 기반 E2E (410개) — Python 의존성만 설치되면 실행됨
❌ 아직 불가: Ubuntu 22.04 신규 머신 1클릭 — Node.js 18+ 단계 누락
❌ 아직 불가: 실제 Telegram 연동 E2E — 봇 토큰 7개 수동 발급 필수
```

**mock 기반 E2E 원클릭 (지금도 가능한 범위)**:
```bash
git clone https://github.com/dragon1086/aimesh && cd aimesh
sudo add-apt-repository ppa:deadsnakes/ppa -y
sudo apt-get install -y python3.11 python3.11-venv curl
bash scripts/setup.sh --yes --skip-verify
.venv/bin/pip install -r requirements-test.txt
.venv/bin/python -m pytest tests/e2e/ -q   # 410개 실행
```
→ **Node.js 설치 + 봇 토큰 없이도 mock 테스트는 통과**

### 질문 2: "새 봇들을 만들어야 하나?"

**YES, 봇 토큰은 수동 발급 필수**.
- PM 봇 1개 + 6개 부서봇 = **최소 7개 @BotFather 발급 필요**
- setup.sh가 자동화할 수 없는 유일한 항목
- 단, mock 기반 E2E 테스트는 봇 토큰 없이도 전부 실행 가능
