## 요약

parse_mode 미지정 호출은 **단순 텍스트 알림** 목적이 대부분이며, 포매팅이 필요 없는 경우에 해당한다.
LLM 응답이나 보고서 내용을 담는 경우는 없으므로, 현재 상태에서 실질적 파싱 오류를 유발하지 않는다.

| 유형 | 건수 | 리스크 |
|------|------|--------|
| 단순 상태 메시지 (✅❌🔄 등) | 30건+ | 🟢 없음 |
| 포매팅 없는 알림/에러 메시지 | 15건+ | 🟢 없음 |
| `/sessions` 등 수동 텍스트 포매팅 | 2건 | 🟡 낮음 |

## 특이사항: progress 중간 메시지

```python
# telegram_relay.py 여러 곳
await progress_msg.edit_text("다 됐어요! ✅")
```
parse_mode 미지정이지만 단순 이모지+텍스트이므로 문제없음.
단, 이전 `edit_text`가 HTML parse_mode로 호출된 경우 동일 메시지 객체에서
parse_mode가 달라 혼용되는 패턴이 있음 — Telegram Bot API는 각 호출마다
parse_mode를 독립 처리하므로 실제 오류는 없으나 코드 일관성 관점에서 개선 권장.
