## ① 제품 개요 및 목적

**TradingAgents**는 실제 트레이딩 회사의 조직 구조(애널리스트 팀 → 리서처 팀 → 트레이더 → 리스크 관리 → 포트폴리오 매니저)를 LLM 에이전트로 재현한 **멀티 에이전트 금융 분석 프레임워크**다.

- **GitHub Stars**: ⭐ 37,900+ (2026-03-22 기준, 급성장 중)
- **라이선스**: Apache 2.0 — 상업적 활용 완전 허용
- **최신 버전**: v0.2.2 (2026년 3월)
- **핵심 포지셔닝**: *"AI가 트레이딩 팀처럼 협업하여 투자 판단을 내린다"*
- **현재 단계**: 연구/프로토타이핑 특화. 실거래 연동 미지원.

### 핵심 가치 제안 (Value Proposition)

| # | 가치 | 설명 |
|---|------|------|
| 1 | **멀티 에이전트 협업** | 단일 LLM 호출 대비 역할 분리된 에이전트 협업으로 분석 품질 향상 |
| 2 | **구조화된 Bull/Bear 토론** | 낙관·비관 리서처 간 디베이트로 편향 감소, 균형 잡힌 의사결정 |
| 3 | **LLM 공급자 독립성** | OpenAI, Google, Anthropic, xAI, Ollama 등 멀티 프로바이더 지원 |
| 4 | **금융 도메인 특화** | 펀더멘탈 · 센티먼트 · 뉴스 · 기술적 분석의 4차원 커버리지 |
| 5 | **연구 재현성** | LangGraph 기반 모듈형 설계로 실험 구성/재현 용이 |
| 6 | **Apache 2.0 오픈소스** | 상업적 파생 제품 제작 가능, GPU 불필요 |

## ② 타깃 사용자 (페르소나)

### 페르소나 A — 퀀트/AI 리서처 (핵심 타깃)
- **프로필**: 석·박사 또는 금융회사 리서치팀, Python 숙련
- **Pain Point**: 단일 LLM 프롬프트로는 복잡한 투자 논리 재현 어려움; 멀티 에이전트 실험 환경 부재
- **Goal**: 멀티 에이전트 트레이딩 전략 연구, 논문 실험 재현
- **TradingAgents 활용**: 에이전트 구성 커스터마이징, LLM 비교 실험

### 페르소나 B — ML/AI 엔지니어 (성장 타깃)
- **프로필**: AI 스타트업 또는 테크 기업 개발자, LLM 실무 경험 보유
- **Pain Point**: 금융 도메인 지식 부족; 에이전트 협업 아키텍처 레퍼런스 없음
- **Goal**: 자체 AI 트레이딩 봇 프로토타입 제작
- **TradingAgents 활용**: TradingAgentsGraph() 클래스 래핑, 자체 데이터 연동

### 페르소나 C — 개인 투자자/금융 개발자 (잠재 타깃)
- **프로필**: 주식/코인 투자 경험 있는 개발자, Python 기초 수준
- **Pain Point**: AI 기반 투자 분석 도구 직접 구축 욕구; 기존 SaaS 도구의 블랙박스 불만
- **Goal**: 개인 투자 판단 보조 도구
- **TradingAgents 활용**: CLI로 종목 분석 실행, 분석 리포트 확인
- **장벽**: API 설정 복잡도, CLI 전용 UX

### 페르소나 D — 기관/핀테크 기업 (고가치 잠재)
- **프로필**: 중소 헤지펀드, 로보어드바이저 스타트업
- **Pain Point**: LLM 기반 분석 POC 필요, 내부 개발 리소스 부족
- **Goal**: 자체 AI 애널리스트 시스템 구축 기반
- **TradingAgents 활용**: Apache 2.0 기반 상업적 Fork, 내부 데이터 통합
- **장벽**: 실거래 미지원, 엔터프라이즈 지원 없음

## ③ 주요 유스케이스

### UC-1. 종목 멀티 에이전트 투자 분석
**시나리오**: 퀀트 리서처가 특정 종목(예: AAPL)에 대해 4개 애널리스트 에이전트를 동시 실행. 펀더멘탈·센티먼트·뉴스·기술적 분석 결과가 Bull/Bear 리서처 토론을 거쳐 최종 Trader 에이전트가 매수/매도 판단 출력.
**가치**: 단일 프롬프트 대비 다각도 분석 자동화

### UC-2. LLM 모델 비교 실험
**시나리오**: AI 연구자가 동일 종목·기간에 대해 GPT-5, Gemini 2.5, Claude 4를 각각 적용하여 투자 판단 품질(수익률, Sharpe ratio) 비교.
**가치**: 금융 도메인 LLM 벤치마킹 플랫폼으로 활용

### UC-3. 에이전트 아키텍처 프로토타이핑
**시나리오**: ML 엔지니어가 TradingAgents를 기반으로 자체 산업 도메인(예: 부동산, 채권)에 맞는 멀티 에이전트 시스템 설계. LangGraph + TradingAgents 구조를 참조 아키텍처로 사용.
**가치**: 멀티 에이전트 설계 패턴 레퍼런스

### UC-4. 교육/데모 시연
**시나리오**: AI 금융 강의나 해커톤에서 LLM 멀티 에이전트 협업 개념을 실시간 CLI 데모로 시연.
**가치**: 개념 가시화, 학습 효과 극대화

### UC-5. 투자 판단 보조 리포트 생성
**시나리오**: 개인 투자자가 관심 종목에 대해 CLI 실행 후 에이전트가 생성한 분석 리포트를 참고하여 투자 판단 보완.
**가치**: 저비용 AI 애널리스트 리포트 (단, 투자 조언 아님 명시 필요)

## ⑤ 개선 로드맵 제언

### 🔴 P0 — 즉시 개선 (핵심 기능 공백)

**[IMP-1] 백테스팅 프레임워크 통합**
- **근거**: 연구 프레임워크를 표방하지만 과거 데이터 기반 전략 검증 기능이 없어 연구 완결성 부족. Backtrader, VectorBT 연동으로 해결 가능.
- **기대 효과**: 연구자 핵심 워크플로우 완성, 논문 재현성 향상
- **난이도**: 중 (외부 라이브러리 연동)

**[IMP-2] 다중 금융 데이터 소스 지원**
- **근거**: Alpha Vantage 단일 의존은 API 한도·데이터 품질 리스크. Yahoo Finance(yfinance), Polygon.io, FRED(거시지표) 추가 필요.
- **기대 효과**: 데이터 신뢰성·다양성 향상, 무료 옵션 제공
- **난이도**: 하 (플러그인 아키텍처로 확장)

### 🟡 P1 — 단기 개선 (사용성·채택률 향상)

**[IMP-3] 웹 UI 대시보드**
- **근거**: CLI 전용 UX는 비개발자(개인 투자자, 비즈니스 사용자) 진입 장벽. 분석 결과 시각화 대시보드로 타깃 확장 가능.
- **기대 효과**: 잠재 사용자 풀 3~5배 확대, GitHub Star 추가 유입
- **난이도**: 중 (Streamlit/Gradio 기반 MVP 빠른 구현 가능)

**[IMP-4] 실거래 브로커 연동 레이어**
- **근거**: 연구에서 실용으로 전환하려는 사용자의 최대 요청. Alpaca(미국), 키움/한국투자증권(국내) 플러그인 구조로 선택적 지원.
- **기대 효과**: 기관/핀테크 채택 급증, 상업적 가치 실현
- **난이도**: 상 (규제·보안 고려 필요)

### 🟢 P2 — 중장기 개선 (제품 완성도)

**[IMP-5] 정량적 리스크 관리 고도화**
- **근거**: 현재 리스크 관리 에이전트가 LLM 기반 정성 평가 수준. VaR, CVaR, 포지션 사이징 등 정량 리스크 모델 통합 필요.
- **기대 효과**: 기관 사용자 신뢰도 향상

**[IMP-6] 포트폴리오 멀티 종목 지원**
- **근거**: 단일 종목 중심 설계로 실제 포트폴리오 운용 시나리오 재현 불가.
- **기대 효과**: 헤지펀드/로보어드바이저 활용 가능성 확대

**[IMP-7] 에이전트 메모리 & 학습 루프**
- **근거**: 현재 에이전트는 세션 간 기억 없음. 과거 분석 결과를 누적 학습하는 메모리 레이어 추가 시 분석 품질 지속 개선 가능.
- **기대 효과**: 장기 사용 시 차별화 가치 창출
