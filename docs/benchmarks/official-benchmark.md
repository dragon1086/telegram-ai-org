## 2. 회귀 탐지 대응 절차서

### 2-1. 회귀 판정 기준

**회귀(Regression)**: 측정된 KPI가 회귀 임계값(하한선) 아래로 하락하는 현상.

```
회귀 심각도 분류:

P0 — 즉시 대응 (Critical)
  조건: KPI가 단일 세션 기준값 이하로 역전되거나, 핵심 KPI 3개 이상 동시 임계 초과
  예: TTD-Multi > 90분 (단일 세션 상한 초과) 또는 QS < 70점

P1 — 당일 대응 (High)
  조건: 핵심 KPI 1-2개가 회귀 임계값 초과, 또는 시스템 기능 저하 확인
  예: CTX-CrossSession < 70% 또는 ERR-Detection < 65%

P2 — 주간 대응 (Medium)
  조건: 보조 KPI 1개가 회귀 임계값 초과, 트렌드 하락 (3회 연속 감소)
  예: TTD-Simple > 9분 또는 QS-Completeness < 80점

P3 — 월간 검토 (Low)
  조건: 단일 KPI 소폭 하락 (임계값 - 5% 이내)
```

### 2-2. 회귀 탐지 절차

```
[1단계] 자동 측정 트리거
  - 주요 배포 후 자동 실행 (섹션 3 참조)
  - 월 1회 정기 측정 (크론 스케줄: 매월 1일 09:00)
  - 수동 트리거: /benchmark-run 명령어 (growth-analysis 스킬 호출)

[2단계] KPI 측정
  - 각 시나리오(SCN-01~04)별 n=3 빠른 실험 실행
  - logs/regression-checks/ 에 결과 저장
  - 이전 공식 기준값과 자동 비교

[3단계] 임계값 비교
  - KPI별 "기준값 - 측정값" 계산
  - 하한선(회귀 임계값) 초과 여부 판정
  - 심각도(P0-P3) 자동 분류

[4단계] 알림 발송
  - P0/P1: 텔레그램 @aiorg_pm_bot 즉시 알림
  - P2: 텔레그램 주간 리포트 포함
  - P3: 월간 벤치마크 리포트 포함

[5단계] 원인 분석
  - 최근 배포 이력 확인 (git log --since="마지막 측정일")
  - 변경된 파일 목록과 하락한 KPI 연관 분석
  - improvement_thresholds.yaml 기준 자가진단

[6단계] 대응 실행
  - P0: 즉시 롤백 또는 긴급 패치
  - P1: 당일 내 원인 패치 + 재측정
  - P2: 주간 스프린트에 개선 태스크 추가
  - P3: 다음 월간 검토 때 확인
```

## 3. 벤치마크 재측정 자동화 대상 목록 및 주기 정의서

### 3-1. 자동화 가능 측정 항목

| KPI | 자동화 가능 | 자동화 방법 | 비고 |
|-----|----------|-----------|------|
| TTD-Simple | ✅ 완전 자동 | ai_org.db 쿼리 | 로그 기반 |
| TTD-Multi | ✅ 완전 자동 | ai_org.db 쿼리 | 로그 기반 |
| TTD-Long | ⚠️ 부분 자동 | 단계별 타임스탬프 합산 | Phase 경계 수동 마킹 필요 |
| QS-Accuracy | ⚠️ 부분 자동 | AC 체크리스트 자동화 가능 항목 | ruff+pytest 자동, 일부 수동 |
| QS-Completeness | ❌ 수동 | 인간 평가 필수 | 섹션 존재 여부는 자동화 가능 |
| QS-Consistency | ❌ 수동 | 의미적 일관성 평가 | LLM 보조 평가 검토 중 |
| PPE-MaxParallel | ✅ 완전 자동 | ai_org.db tasks 스냅샷 | 5분 주기 폴링 |
| PPE-TimeReduction | ✅ 완전 자동 | 태스크 타임라인 계산 | |
| CTX-CrossSession | ⚠️ 부분 자동 | 키워드 매칭 검증 | 정확도 검증은 수동 |
| CTX-LongTask | ⚠️ 부분 자동 | 수치 인용 정확도 스크립트 | |
| ERR-Detection | ✅ 완전 자동 | 훅 로그 분석 | log-agent.sh 필요 |
| ERR-Recovery | ✅ 완전 자동 | 훅 + 에이전트 로그 분석 | |
| ERR-MTTR | ✅ 완전 자동 | 오류 시각 ~ 복구 시각 계산 | |

**완전 자동화율**: 7/13 (54%) — 훅 시스템(GAP-1) 완전 해소 후 9/13 (69%)로 상승 예상

### 3-2. 벤치마크 재측정 주기 정의

| 트리거 유형 | 주기/조건 | 실행 범위 | 담당 |
|-----------|---------|---------|------|
| **정기 측정** | 매월 1일 09:00 | 전체 KPI (SCN-01~04) | 자동 (크론) |
| **배포 후 측정** | 주요 배포 완료 후 1시간 이내 | 핵심 KPI 5개 (빠른 버전) | 자동 (훅 트리거) |
| **긴급 측정** | P0/P1 인시던트 발생 시 | 이상 감지 KPI + 연관 KPI | 수동 (/benchmark-run) |
| **분기 전체 검토** | 매 분기 마지막 주 | 전체 KPI + 추세 분석 | 성장실 PM |
