## 3. 선 스타일 가이드

| 선 유형 | 표기 | 의미 |
|--------|------|------|
| 실선 두꺼운 (`──`) | 필수 관계, FK 참조 | 물리적 연결 |
| 실선 얇은 (`──`) | 일반 흐름 방향 | 데이터 흐름 |
| 점선 (`┄┄`) | 선택적 관계, 논리적 참조 | NULL 허용, 미래 연결 |
| 굵은 화살표 (`═══►`) | 주 흐름 경로 | 권장 경로 |
| 가는 화살표 (`───►`) | 대안 경로 | 폴백, 예외 처리 |

---

## 4. 카디널리티 표기 규칙

| 기호 | 의미 | ERD 표기 |
|------|------|---------|
| `||` | 정확히 1 | ONE |
| `|o` | 0 또는 1 | ZERO_OR_ONE |
| `}o` | 0 또는 N | ZERO_OR_MANY |
| `}|` | 1 또는 N | ONE_OR_MANY |

## 6. 접근성 (WCAG 2.1 AA) 준수 사항

| 항목 | 기준 | 준수 여부 | 비고 |
|------|------|---------|------|
| 색상 대비율 (일반 텍스트) | 4.5:1 이상 | ✅ | 다크 배경 + 화이트 텍스트 = 7:1 |
| 색상 대비율 (대형 텍스트) | 3:1 이상 | ✅ | 헤더 텍스트 기준 |
| 색상만으로 정보 전달 금지 | 형태+레이블 병용 | ✅ | 아이콘 + 텍스트 레이블 병기 |
| 텍스트 크기 | 최소 14px (본문) | ✅ | 16px 기본, 12px 캡션 |
| 포커스 표시 | 2px 이상 외곽선 | ✅ | 인터랙티브 요소 포커스 링 |

> **색맹 고려**: 레드-그린 색맹 대응을 위해 색상 외 형태(실선/점선/굵기)와 텍스트 레이블을 병용함. 알라이즈 레드 사용 시 반드시 "❌ ERROR", "⚠️ WARN" 등 아이콘 병기.
