# 텔레그램 이스케이프 필요 문자 목록

> 출처: Telegram Bot API 공식 문서 (https://core.telegram.org/bots/api#formatting-options)

## MarkdownV2 — 이스케이프 필수 문자 (총 18개)

다음 문자들은 MarkdownV2 모드에서 **반드시 백슬래시(`\`)로 이스케이프** 해야 합니다.
서식 지정에 쓰이지 않는 일반 텍스트 위치에서도 예외 없이 적용됩니다.

| 문자 | 이스케이프 형태 | 비고 |
|------|--------------|------|
| `_`  | `\_`  | italic/underline 구분자 |
| `*`  | `\*`  | bold 구분자 |
| `[`  | `\[`  | 링크/mention 시작 |
| `]`  | `\]`  | 링크/mention 끝 |
| `(`  | `\(`  | URL 시작 |
| `)`  | `\)`  | URL 끝 |
| `~`  | `\~`  | 취소선 구분자 |
| `` ` `` | `` \` `` | 코드 구분자 |
| `>`  | `\>`  | 인용구 구분자 |
| `#`  | `\#`  | 헤더 (미지원이나 예약 문자) |
| `+`  | `\+`  | 예약 문자 |
| `-`  | `\-`  | 목록/수평선 구분자 |
| `=`  | `\=`  | 예약 문자 |
| `\|` | `\|`  | 스포일러/표 구분자 |
| `{`  | `\{`  | 예약 문자 |
| `}`  | `\}`  | 예약 문자 |
| `.`  | `\.`  | 예약 문자 (숫자 목록에서도 필수) |
| `!`  | `\!`  | 예약 문자 |

### 추가 규칙
- URL 내부의 `)` 및 `` ` `` 도 이스케이프 필요
- `\` 자체도 `\\` 로 이스케이프 필요
- pre/code 블록 내부에서는 `` ` `` 과 `\` 만 이스케이프

## HTML parse_mode — 이스케이프 필수 문자 (총 3개)

| 문자 | HTML 엔티티 | 설명 |
|------|-----------|------|
| `&`  | `&amp;`   | 앰퍼샌드 |
| `<`  | `&lt;`    | 꺾쇠 열기 |
| `>`  | `&gt;`    | 꺾쇠 닫기 |

### 현재 코드베이스 처리 방식 (`core/telegram_formatting.py`)

```python
def escape_html(text: str) -> str:
    return text.replace("&", "&amp;").replace("<", "&lt;").replace(">", "&gt;")
```

**✅ 올바르게 구현됨** — 3가지 HTML 엔티티를 모두 처리하고 있습니다.

## 결론

- **MarkdownV2**: 18개 특수문자를 빠짐없이 이스케이프해야 하므로 LLM 출력 후처리에 부적합
- **HTML**: 3개 문자만 이스케이프하면 되므로 프로그래밍 방식의 메시지 생성에 적합
- **현재 코드베이스**: HTML 방식 채택 ✅ (올바른 선택)
