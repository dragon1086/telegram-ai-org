## HTML parse_mode 이스케이프 규칙

**필수 이스케이프 문자: 3개**

| 원문 문자 | HTML 엔티티 | 이유 |
|-----------|-------------|------|
| `&` (앰퍼샌드) | `&amp;` | HTML 엔티티 구분자 |
| `<` (꺽쇠 열기) | `&lt;` | 태그 시작 구분자 |
| `>` (꺽쇠 닫기) | `&gt;` | 태그 종료 구분자 |

**현재 코드베이스 처리 방식** (`core/telegram_formatting.py`):
```python
def escape_html(text: str) -> str:
    return text.replace("&", "&amp;").replace("<", "&lt;").replace(">", "&gt;")
```
✅ 정확히 구현됨

## MarkdownV2 parse_mode 이스케이프 규칙

**필수 이스케이프 문자: 18개**

모든 특수문자는 일반 텍스트로 사용 시 `\` 접두사 필수.

| 문자 | 이스케이프 | LLM 출력 빈도 | 위험도 |
|------|------------|--------------|--------|
| `_` | `\_` | 매우 높음 (snake_case) | 🔴 높음 |
| `*` | `\*` | 높음 (볼드 마크다운) | 🔴 높음 |
| `[` | `\[` | 보통 | 🟡 중간 |
| `]` | `\]` | 보통 | 🟡 중간 |
| `(` | `\(` | 보통 | 🟡 중간 |
| `)` | `\)` | 보통 | 🟡 중간 |
| `~` | `\~` | 낮음 | 🟢 낮음 |
| `` ` `` | `` \` `` | 높음 (코드) | 🔴 높음 |
| `>` | `\>` | 보통 | 🟡 중간 |
| `#` | `\#` | 높음 (헤더) | 🔴 높음 |
| `+` | `\+` | 보통 | 🟡 중간 |
| `-` | `\-` | 매우 높음 (목록) | 🔴 높음 |
| `=` | `\=` | 보통 | 🟡 중간 |
| `\|` | `\|` | 높음 (테이블) | 🔴 높음 |
| `{` | `\{` | 낮음 | 🟢 낮음 |
| `}` | `\}` | 낮음 | 🟢 낮음 |
| `.` | `\.` | 매우 높음 (모든 문장) | 🔴 높음 |
| `!` | `\!` | 높음 | 🔴 높음 |

> ⚠️ **결론**: LLM 출력에는 `-`, `_`, `*`, `#`, `.`, `!` 등이 매우 빈번하게 등장한다.
> MarkdownV2를 사용하려면 전체 텍스트를 정규식으로 전처리하는 별도 이스케이퍼가 필수이며,
> 이를 누락하면 텔레그램 API가 `400 Bad Request`를 반환한다.

## 현재 코드베이스 이스케이프 처리 현황

### ✅ 올바르게 처리 중
- `core/telegram_formatting.py`의 `escape_html()` — `&`, `<`, `>` 이스케이프
- `markdown_to_html()` 내부에서 코드블록 먼저 추출 → 나머지 텍스트 이스케이프 → 마크다운 변환 순서 준수

### ⚠️ 주의 필요
- f-string으로 직접 구성하는 HTML 메시지에서 `d.get('direction')` 같은 동적 데이터가 이스케이프 없이 삽입되는 경우:
  ```python
  # 예: core/telegram_relay.py:3650
  f"• 방향성: {d.get('direction','미설정')}\n\n"
  # → direction 값에 &, <, > 포함 시 HTML 파싱 오류 가능
  ```
- 이 경우 `escape_html()` 래핑이 필요:
  ```python
  f"• 방향성: {escape_html(d.get('direction','미설정'))}\n\n"
  ```
