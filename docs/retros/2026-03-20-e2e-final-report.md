## 아키텍처 인사이트

이번 E2E 테스트를 통해 확인된 시스템 아키텍처:

```
사용자 메시지 → PM 봇 (aiorg_pm_bot)
                    ↓
              NL 분류기 → 라우팅 결정
                    ↓
         [직접답변] 또는 [배분 모드]
                    ↓ (배분 모드)
         cross_org_bridge → worker 봇
                    ↓ (비동기 실행)
         Claude Code / Codex runner
                    ↓
         PM 봇이 결과 합성 후 최종 응답
```

Worker 봇은 Telegram 그룹에 직접 응답하지 않고,
PM 봇이 결과를 수신하여 합성 후 그룹에 응답.
