## 정성적 코멘트 (접근법별)

### 벡터 DB — 20/30
**강점**: 시맨틱 검색에서 압도적. "봇 재기동"을 쿼리하면 "request_restart.sh 사용"이 연결됨. Chroma는 단일 Python 패키지로 설치, 파일 기반 저장이 가능해 로컬 적합.

**약점**: 마크다운 파일이 벡터로 변환되면 원본 파일과 이원화됨. "ACT-5가 task T-260을 구현함" 같은 관계 정보는 저장 불가. 임베딩 모델(로컬 Ollama or API) 의존.

**우리 환경 적합성**: Chroma + 로컬 임베딩(nomic-embed-text via Ollama)이면 외부 API 없이 운영 가능. 그러나 단독으로는 R1(관계 표현) 해결 못함.

---

### 그래프 DB — 18/30
**강점**: 관계 표현력이 독보적. Graphiti의 bi-temporal 모델은 R3(시간 유효성)를 완벽 해결. neo4j-labs/agent-memory는 MCP로 Claude와 직접 연결 가능.

**약점**: Neo4j는 Docker + JVM 메모리 부담. 소규모 운영(현재 3개 메모리 파일 수준)에 오버엔지니어링. NetworkX(Python 인메모리)는 가볍지만 재시작 시 초기화.

**우리 환경 적합성**: 장기적으로는 최적이지만 단기 구현 비용이 높다. Graphiti는 Neo4j 필수 — 현재 인프라에 Docker Neo4j 추가 필요.

---

### JSON-LD 메타데이터 — 23/30 ★ 최고점
**강점**: 기존 마크다운 파일을 그대로 유지하며 YAML frontmatter만 추가. 개발 공수 최소. 인간이 읽을 수 있는 파일 + 기계가 파싱할 수 있는 메타데이터 공존. 현재 project_pending_tasks.md가 이미 부분적으로 YAML frontmatter 사용 중.

**약점**: 시맨틱 검색 불가. 관계 표현은 "related: [id, relation]" 형태로 가능하지만 graph traversal 없이는 2단계 이상 연결 추적 어려움.

**우리 환경 적합성**: 현재 구조와 가장 자연스럽게 통합. 빠른 시작 가능. 단, 검색 품질 향상을 위해 벡터 DB나 태그 인덱스와 반드시 조합 필요.

---

### 태그 기반 매핑 — 21/30
**강점**: 구현이 단순 (JSON 인덱스 파일 하나). 마크다운 frontmatter에 tags 필드 추가로 즉시 적용. 계층 태그로 봇별/도메인별 필터링 가능.

**약점**: 태그 폭발 관리 필요. 시맨틱 검색 불가. 관계 방향성 표현 약함.

**우리 환경 적합성**: JSON-LD 메타데이터의 tags 필드와 자연스럽게 결합. 단독으로는 약하지만 다른 접근법의 보완재로 필수적.

## 하이브리드 조합 시나리오

### 시나리오 1: "마크다운 퍼스트 + 메타데이터 인덱스" (즉시 적용 가능, 권장)
```
구성: 마크다운 파일 + YAML frontmatter(JSON-LD) + 태그 역색인 JSON + SQLite 메타 인덱스
검색: 태그/필터 → 후보 파일 → 파일 본문 LLM 처리
관계: frontmatter의 related[] 필드
시간: valid_from / valid_until / updated_at 필드
```
- 장점: 기존 파일 구조 유지, 외부 의존성 없음, 1-2일 구현
- 단점: 시맨틱 검색 없음, 대규모 파일 시 검색 지연

### 시나리오 2: "마크다운 퍼스트 + 벡터 레이어" (중기 적용)
```
구성: 시나리오 1 + Chroma(로컬) + 로컬 임베딩(nomic-embed-text)
저장: 마크다운 파일 변경 시 자동 re-embed → Chroma 업데이트
검색: 시맨틱 쿼리 → Chroma top-K → 원본 파일 fetch
메타 필터: Chroma where 절로 frontmatter 메타데이터 필터링
```
- 장점: R4(시맨틱 검색) 해결, 로컬 운영 가능
- 단점: Ollama 필요, 초기 임베딩 시간

### 시나리오 3: "풀 그래프 메모리" (장기 목표)
```
구성: Graphiti + Neo4j(Docker) + 벡터 임베딩
저장: 대화/태스크 → LLM 분석 → 지식 그래프 노드/엣지 자동 생성
검색: 시맨틱 + 그래프 트래버설 복합
시간: bi-temporal 모델 (사건시간 + 수집시간)
```
- 장점: 모든 요구사항(R1~R7) 충족
- 단점: 구현 공수 큼, Neo4j 인프라 필요, 안정화 기간 필요
