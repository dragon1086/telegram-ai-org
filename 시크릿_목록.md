# 시크릿 목록

GitHub Actions CI/CD에서 사용하는 저장소 secret 정리본이다.

| Secret | 워크플로우 | 필수 여부 | 용도 |
|---|---|---|---|
| `ANTHROPIC_API_KEY` | `e2e-test.yml` | 선택 | `claude-code` 매트릭스 인증 컨텍스트 |
| `OPENAI_API_KEY` | `e2e-test.yml` | 선택 | `codex` 매트릭스 인증 컨텍스트 |
| `GEMINI_OAUTH_CREDS` | `e2e-test.yml` | 선택 | `gemini-cli` OAuth JSON을 `~/.gemini/oauth_creds.json`으로 복원 |
| `PYPI_API_TOKEN` | `publish-pypi.yml` | 필수 | PyPI 업로드 토큰 |
| `DOCKER_USERNAME` | `docker-build-push.yml` | 필수 | Docker Hub 로그인 계정 |
| `DOCKER_PASSWORD` | `docker-build-push.yml` | 필수 | Docker Hub access token 또는 비밀번호 |

등록 위치:

1. GitHub 저장소 `Settings`
2. `Secrets and variables`
3. `Actions`
4. `New repository secret`

운영 메모:

- `e2e-test.yml`은 secret이 없어도 테스트 자체는 계속 실행된다.
- `publish-pypi.yml`과 `docker-build-push.yml`은 secret이 없으면 배포 단계에서 실패한다.
- Gemini는 로컬 운영과 CI 모두 OAuth 기준으로 유지하고, CI에서만 `GEMINI_OAUTH_CREDS` secret을 파일로 복원한다.
